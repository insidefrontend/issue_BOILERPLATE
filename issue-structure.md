# イベント毎に作成するディレクトリ、ファイル構造について

Inside Frontendではイベント毎に`issue #`レポジトリを作成し、イベントの管理を行う。  
本ドキュメントでは`issue #`のディレクトリ、ファイル構造の仕様について説明する。

## Issue

- `/backyard`　<= イベント開催裏方系ドキュメント
	- REAMDME.md
		- 開催日
		- 募集開始日
		- イベント募集ページURL
		- 会場
			- 最大収容数
			- 費用
		- スタッフ/サポート人員について
- `/sessions`　<= スピーカ、セッション系ドキュメント
	- `{$セッション名(英語)}`
		- `/slide`
			- PDF
			- スライド内のURLリスト.md
		- README.md
			- スピーカ名
				- SNSs
				- 所属
			- タイトル
			- 概要(100文字程度)
			- アウトライン
- `/booths` <= ブース系ドキュメント
	- README.md
		- ブース #
			- トピック名
				- 概要(100文字程度)
			- ブース・エキスパート
				- 氏名:
				- 所属:
				- SNSs:
